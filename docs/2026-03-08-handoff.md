# WordCore Handoff

> Date: 2026-03-08

## Current Product Direction

WordCore is being reshaped into an ultra-minimal imitation-writing tool for the core 3000 English words.

The intended learning loop is:

1. Read the word, definition, and reference sentence together
2. Write one similar sentence with a small change
3. Run AI-based sentence checking on demand
4. Save the learner sentence
5. Review and repeat until the word is genuinely usable

The user strongly prefers:

- minimal, quiet UI
- no decorative clutter
- low cognitive load
- sentence-writing as the main action

## What Is Implemented

### Data and Study Flow

- Word bank normalization and filtering are in place
- Study selection is no longer fully random; it prefers unfinished words and avoids instant repetition
- Learner drafts are auto-saved per word in localStorage
- Word bank page can show learner-written sentences
- CSV export is available from the word bank page

### AI Sentence Check

- `Self-check` now calls Gemini from the frontend
- AI feedback is structured as:
  - acceptable / needs revision
  - short grammar or naturalness feedback
  - suggested revision
- Runtime service lives in:
  - `/Users/dengshu/development/wordcore/src/services/sentenceCheck.js`

### Desktop Layout

- Desktop now uses a left rail + main workspace shell
- Study page has been reworked several times to reduce visual fragmentation
- Latest structure:
  - top header with short task framing
  - one unified word card containing word, part of speech, definition, and reference sentence
  - lower left writing area
  - lower right AI result area

## Latest UI Decisions

These decisions reflect the latest user feedback and should be preserved unless there is a strong reason to change them:

1. Reference sentence should live in the same top card as the word and definition
2. Learner sentence area and AI result should both remain visible on the same screen
3. AI result should not be embedded inside the reference block
4. The left rail should stay present, but lighter and quieter than earlier versions
5. The Study page should feel like a focused work surface, not a dashboard

## Important Environment Note

Frontend AI checking reads:

- `VITE_GEMINI_API_KEY`

The offline word generation script reads:

- `GEMINI_API_KEY`

A previous confusion happened because only `GEMINI_API_KEY` was set, while the frontend needed `VITE_GEMINI_API_KEY`.

After editing `.env`, the Vite dev server must be restarted.
